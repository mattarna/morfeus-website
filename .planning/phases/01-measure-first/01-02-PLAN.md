    Create docs/geo/ directory if it does not exist. Write docs/geo/search-console-bing-setup.md with the following structure:

    Title: "Google Search Console + Bing Webmaster Tools — Setup e Verifica"

    Section 1 "Perché è prioritario" — three bullet points:
    - Google Search Console: mostra le query organiche reali, le pagine indicizzate vs escluse, gli errori di copertura. Senza GSC non sappiamo quali pagine Google ha effettivamente indicizzato.
    - Bing Webmaster Tools: ChatGPT Search (chatgpt.com) e Microsoft Copilot si appoggiano in larga parte all'indice di Bing. Un sito non verificato su BWT non ottiene l'AI Performance Report (dati di impression AI unici per piattaforma). Il Bing index è il secondo punto di ingresso LLM più importante dopo Google.
    - Sitemap: https://morfeushub.com/sitemap.xml — già esistente e funzionante. Va solo sottomessa su entrambe le piattaforme.

    Section 2 "PARTE A — Google Search Console" — subsections:

    2.1 "Creazione/accesso alla property" — steps:
    1. Vai su https://search.google.com/search-console/
    2. Clicca "Add property" → scegli "Domain" (non "URL prefix") → inserisci: morfeushub.com
    3. [HUMAN-ACTION] Google propone un record TXT da aggiungere al DNS. Aggiungilo nel pannello del tuo registrar (es. Namecheap, GoDaddy, Cloudflare): Tipo: TXT | Host: @ | Valore: google-site-verification=XXXXXXXX | TTL: default
    4. Clicca "Verify" su GSC. La propagazione DNS può richiedere da pochi minuti a 48 ore.
    5. Se il registrar non supporta record TXT al root (@), usa il metodo alternativo "HTML file": scarica il file google-verification-XXXX.html e carica nella root del progetto come public/google-verification-XXXX.html, poi commetti e fai deploy su Vercel prima di cliccare Verify. Questo è il metodo preferito se hai accesso al codice ma non al DNS.

    2.2 "Invio sitemap" — steps:
    1. Una volta verificata la property, vai su Sitemaps (menu sinistro).
    2. Nel campo "Add a new sitemap", inserisci: sitemap.xml (GSC aggiunge automaticamente il dominio)
    3. Clicca "Submit". Lo stato passerà da "Pending" a "Success" entro poche ore/giorni.
    4. Verifica: la colonna "Discovered URLs" mostra il numero di URL trovati nella sitemap.

    2.3 "Baseline di copertura" — steps:
    1. Dopo 3-7 giorni, vai su Coverage (o "Pages" nel nuovo GSC).
    2. Annota il numero di pagine nello stato "Indexed" per i locale /it/ e /en/.
    3. Verifica che non compaiano URL con /funnel-internal/ nello stato "Indexed" — se compaiono, è un segnale che il noindex middleware non è attivo (da risolvere in Phase 2).
    4. Salva uno screenshot della pagina Coverage in docs/geo/baselines/YYYY-MM/gsc-coverage-baseline.png.

    Section 3 "PARTE B — Bing Webmaster Tools" — subsections:

    3.1 "Creazione account e verifica" — steps:
    1. Vai su https://www.bing.com/webmasters/
    2. Accedi con l'account Microsoft di Morfeus (o crea un account Microsoft se non esiste).
    3. [HUMAN-ACTION] Clicca "Add your site" → inserisci https://morfeushub.com
    4. Bing offre tre metodi di verifica. Usa il metodo XML file (più semplice per chi ha accesso al codice):
       a. Scarica il file BingSiteAuth.xml che Bing genera per il tuo sito.
       b. Inseriscilo in public/BingSiteAuth.xml nel progetto.
       c. Commetti il file: git add public/BingSiteAuth.xml && git commit -m "chore(seo): add Bing WMT verification file"
       d. Fai push su Vercel (o aspetta il deploy automatico da main).
       e. Torna su BWT e clicca "Verify".
    5. Alternativa: metodo CNAME DNS (se preferisci non toccare il codice) — aggiungi il record CNAME fornito da Bing nel tuo registrar.
    6. [HUMAN-ACTION] Conferma verifica riuscita: lo stato del sito passa da "Pending" a "Verified".

    3.2 "Invio sitemap" — steps:
    1. Vai su Sitemaps nel pannello BWT.
    2. Clicca "Submit sitemap" → inserisci: https://morfeushub.com/sitemap.xml
    3. Clicca "Submit". Lo stato passerà a "Success" entro 24-48 ore.

    3.3 "AI Performance Report" — steps:
    1. Una volta verificata e con sitemap attiva, cerca nel menu il link "AI Performance Report" (o "Reports → AI Performance").
    2. Il report mostra le impression generate da Copilot e dai risultati AI di Bing per le pagine del sito.
    3. All'inizio i dati possono essere zero o molto bassi — è normale. Il report diventa significativo dopo 30+ giorni con traffico Bing.
    4. Salva uno screenshot baseline in docs/geo/baselines/YYYY-MM/bing-ai-report-baseline.png.

    Section 4 "Post-setup checklist" — a Markdown checklist (- [ ] items):
    - [ ] GSC property verificata (record TXT nel DNS o file HTML in public/)
    - [ ] sitemap.xml sottomessa su GSC → stato "Success"
    - [ ] Bing WMT account creato e sito verificato
    - [ ] BingSiteAuth.xml committato e su Vercel (se metodo XML usato)
    - [ ] sitemap.xml sottomessa su BWT → stato "Success"
    - [ ] Screenshot coverage GSC salvato in docs/geo/baselines/
    - [ ] Screenshot AI Performance Report BWT salvato in docs/geo/baselines/
    - [ ] Questo file aggiornato con la data di completamento di ogni step

    Section 5 "Cadenza post-setup" — bullet list:
    - Settimanale: controlla GSC Coverage per nuove pagine indicizzate/escluse; controlla BWT Crawl per errori.
    - Mensile: esporta i dati Performance (impressioni, click, CTR) per locale IT/EN; confronta con mese precedente.
    - Dopo ogni deploy significativo: re-submit sitemap su GSC se la sitemap è cambiata (Phase 2+ cambierà la sitemap).

    Section 6 "Note tecniche" — two bullets:
    - La sitemap generata da Next.js è disponibile a runtime su https://morfeushub.com/sitemap.xml e viene aggiornata automaticamente ad ogni build. Non è un file statico — non committarla in git.
    - Il file BingSiteAuth.xml è l'unico file di verifica che va committato in public/. Il file google-verification-XXXX.html va committato solo se si usa il metodo HTML file (alternativa al DNS TXT).
