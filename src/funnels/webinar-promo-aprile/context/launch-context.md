# CONTEXT STRATEGY — Launch 1 Full Funnel (Webinar → Offer → Course Welcome)

Use this doc to design + write ONLY the pages for Launch 1.
Primary goal: **sell the course**.
Key principle: one clear next step per page, minimal cognitive load.

## 0) Funnel scope (pages to build)
1) Webinar Registration Page (Opt-in)
2) Webinar Thank You Page (post opt-in)
3) AI Playground Join (embedded in TY or separate join page)
4) Webinar "Offer Transition" page block (optional, if needed as bridge)
5) Sales Page (course checkout page or sales letter page → checkout)
6) Post-Purchase Thank You / Course Welcome Page

## 1) Funnel narrative (simple, non-hype)
- Opt-in: promise the webinar outcome + include AI Playground access.
- Webinar: deliver value + introduce the paid course as the execution path.
- Sales page: convert to purchase with clear promise, proof, offer stack, risk reversal.
- Post-purchase: confirm, set expectations, reduce refunds, drive first action.

## 2) Core promise (keep consistent)
Core promise structure:
- **Build your first AI Employee in 30 days**
Meaning:
- deploy a working AI system on a real recurring task, not "learn AI".

## 3) ICP for this funnel (tight)
Included:
- professionals, freelancers, small teams with repetitive work
- wants time back, stability, and a practical system
Excluded:
- easy money seekers
- dev/coding learners
- curiosity-only users

## 4) Main conversion levers (what must be present across pages)
- Outcome clarity (what they get)
- Timebox clarity (30 days)
- Ease-to-start (no-code / step-by-step, if true)
- Proof via assets and examples (not generic claims)
- Objection handling (time, technical, tool volatility, "I tried ChatGPT")
- Risk reversal (refund/guarantee terms if offered)
- One primary CTA per page/section

## 5) Product to sell (Launch 1 course)
Product type: **infoproduct course** (practical, implementation-focused).
Positioning:
- "AI Employee" builder system for operators
- assets-first delivery (templates, SOPs, checklists, workflows)
Transformation:
- from "using AI occasionally" → to "having an AI Employee that runs a recurring workflow"

What the buyer expects (must be explicit on sales page):
- step-by-step build process
- concrete deliverables per module
- examples/use cases to replicate
- support environment = AI Playground as execution hub (if included)

## 6) Offer stack (how to present it)
Core elements to include (adapt wording to final curriculum):
- Main course (structured path to build 1 AI Employee)
- Templates + SOPs + checklists (production assets)
- Use case library / examples (by role: marketing/sales/ops/admin)
- Access to AI Playground (as workspace/resources hub, not "community")

Pricing:
- Do not invent numbers here. Use placeholders until final price is provided:
  - PRICE: [€___]
  - PAYMENT: [one-time / split-pay if allowed]

## 7) Proof system for sales conversion
Sales page must show at least:
- screenshots of assets/templates
- 1–3 concrete "AI Employee" examples (what it does, inputs, outputs)
- before/after workflow snapshot
Optional if available:
- case snippets (role + starting point + result)

Avoid:
- vague testimonials
- inflated or unverifiable claims

## 8) Objections to handle on sales page (must-have FAQ)
- "I'm not technical"
- "I don't have time"
- "Tools change too fast"
- "Will this work for my role?"
- "I'm afraid it's just theory"
- "What if I get stuck?"
Answer pattern:
- system > tools
- minimal weekly time + clear steps
- role-based examples
- assets-first, not theory
- support via hub + troubleshooting resources
